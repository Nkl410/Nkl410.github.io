Context

DURA is an internal engineering platform developed as part of my current role at eMotors (Stellantis), designed to structure and automate durability analytics. The goal is to move from manual, file-based workflows to standardized pipelines with full traceability, reproducible results, and clear dashboards for engineering teams.

Problems addressed

  • Manual analysis steps and inconsistent outputs across teams
  • Heterogeneous input files and hard-to-reuse processing scripts
  • Limited traceability (who ran what, with which parameters)
  • No centralized view of mission profiles, results, and status

Objectives

  • Automate mission profile processing (standard pipeline)
  • Store inputs, outputs, and metadata for full traceability
  • Provide dashboards for quick validation and reporting
  • Design a scalable architecture to add future modules

My role

I led the project end-to-end: functional analysis, technical architecture, data model, first module implementation, and stakeholder alignment.

Platform modules

The platform is designed as a set of independent modules, sharing a common data model and consistent workflows.

Mission Profile (MP) module

Centralizes mission profile inputs, runs standardized processing, stores outputs and metadata, and provides dashboard-ready summaries.

Architecture overview

Designed to be scalable and industrial-friendly: clear separation between ingestion, processing, storage, and visualization layers.

Data layer

  • Structured entities: groups, files, runs, results
  • History tables for audit trail
  • Metadata and parameters saved with each run

Processing layer

  • ETL pattern: ingest → clean → compute → store
  • Deterministic outputs for reproducibility
  • Validation rules to ensure data quality

App layer

  • Dashboard-oriented KPIs and visuals
  • Clear navigation per module
  • Export-friendly results
